# 需求标准化模块 - 产品需求文档

## 1. 模块概述

### 1.1 模块定位
需求标准化是智能测试用例平台的入口模块，负责将用户输入的原始需求（文本或文档）通过AI转换为标准化的需求规格说明书，并拆分为可执行的单个需求项，为后续测试设计提供结构化输入。

### 1.2 核心流程

```
需求录入 → 文档标准化 → 需求拆分 → 进入测试设计
```

### 1.3 模块价值
- **规范化**：将非结构化的原始需求转换为符合IEEE 830标准的规格说明书
- **智能化**：AI辅助头脑风暴，帮助用户完善和丰富需求内容
- **可追溯**：版本管理 + 历史记录，完整保留需求演进过程
- **高质量**：自动质量评分，引导用户产出高质量需求文档

## 2. 页面布局

### 2.1 整体结构

```
┌──────────────────────────────────────────────────────┐
│  左侧边栏（历史记录）    │  主内容区                    │
│                         │  ┌─────────────────────┐   │
│  [+ 新建需求]           │  │  步骤导航 1→2→3     │   │
│                         │  ├─────────────────────┤   │
│  历史记录列表            │  │                     │   │
│  - 记录1               │  │  步骤内容区          │   │
│  - 记录2               │  │                     │   │
│  - 记录3               │  │                     │   │
│                         │  └─────────────────────┘   │
└──────────────────────────────────────────────────────┘
```

### 2.2 左侧边栏 - 历史记录

| 功能点 | 描述 |
|--------|------|
| 新建需求入口 | 顶部「+ 新建需求」按钮，点击重置所有步骤，开始新的标准化流程 |
| 历史列表 | 展示历史需求记录，显示标题和日期 |
| 选中查看 | 点击某条历史记录，加载完整数据覆盖当前工作区 |
| 数据保留 | 每条记录保留：原始需求、标准化文档、拆分需求、测试点、测试用例 |

### 2.3 步骤导航

三个步骤以 `① → ② → ③` 形式横向排列在内容区顶部：

| 步骤 | 名称 | 跳转条件 |
|------|------|----------|
| 步骤1 | 需求录入 | 始终可访问 |
| 步骤2 | 文档标准化 | 完成标准化后解锁 |
| 步骤3 | 需求拆分 | 完成拆分后解锁 |

**步骤间自由跳转**：用户可点击任意已解锁步骤进行跳转。当修改前面步骤的数据时，后面步骤的数据自动清空以保证数据一致性。

## 4. 步骤2 - 文档标准化

### 4.1 标准文档模板

标准化文档基于 **IEEE 830-1998** 标准，包含以下固定章节框架：

| 章节 | 子章节 |
|------|--------|
| 1. 引言 | 1.1 编写目的、1.2 项目背景、1.3 术语定义 |
| 2. 需求概述 | 2.1 业务目标、2.2 用户角色、2.3 核心业务流程 |
| 3. 功能需求 | 3.1 功能模块一、3.2 功能模块二（可扩展） |
| 4. 非功能需求 | 4.1 性能需求、4.2 安全性需求、4.3 可用性需求、4.4 兼容性需求 |
| 5. 约束条件 | 5.1 技术约束、5.2 业务约束、5.3 法规约束 |
| 6. 异常场景处理 | 6.1 异常场景一、6.2 异常场景二（可扩展） |

**模板规则**：
- 章节框架不可删除，保证文档结构标准化
- 各章节内容可为空，用户可按需填写
- 模板以独立文件形式保存，便于维护和升级

### 4.2 标准文档预览

| 功能点 | 描述 |
|--------|------|
| 展示形式 | Markdown 格式在模拟浏览器窗口中预览 |
| 可编辑 | 用户可直接在文本框中编辑文档内容 |
| AI生成标识 | 标题旁显示「AI 生成」绿色标签 |
| 提示语 | 底部显示「可直接编辑文档内容，或使用AI头脑风暴与AI进行多轮对话帮您调整」 |

### 4.3 文档质量评分

标准化完成后自动对文档进行三维度质量评估：

| 维度 | 权重 | 检测内容 |
|------|------|----------|
| 完整性 | 45% | 检查6个必填章节是否有实质内容（非仅占位符） |
| 清晰度 | 35% | 检测模糊词汇（大概/可能/差不多等）、是否有量化指标、是否有示例 |
| 一致性 | 20% | 检测术语混用（登录/登陆、账号/帐号）、矛盾描述 |

**评分展示**：
- 环形仪表盘显示总分，颜色随等级变化
- 三个维度各一条进度条
- 评分等级：良好（≥80 绿色）、一般（60-79 黄色）、需改进（<60 红色）
- 改进建议列表（最多3条），附带「使用AI头脑风暴改进文档」快捷入口

### 4.4 操作按钮

| 按钮 | 样式 | 行为 |
|------|------|------|
| AI头脑风暴 | 蓝色文字按钮 | 打开AI头脑风暴弹窗 |
| 导出 | 灰色边框下拉按钮 | 展开导出格式选择菜单 |
| 需求拆分 | 蓝色实心按钮 | 将标准化文档拆分为单个需求，完成后跳转步骤3 |

## 5. AI头脑风暴

### 5.1 弹窗布局

```
┌─────────────────────────────────────────────────────┐
│  AI头脑风暴                              [✕ 关闭]   │
├──────────────────────┬──────────────────────────────┤
│  对话区（左 50%）     │  预览区（右 50%）             │
│                      │  ┌──────────────────────┐   │
│  [对话消息列表]       │  │ 版本选择器  [恢复此版本] │   │
│                      │  ├──────────────────────┤   │
│  AI: 我建议将...     │  │ 🟢新增 🟡修改 🔴删除  │   │
│  [采纳] [拒绝]       │  ├──────────────────────┤   │
│                      │  │                      │   │
│  用户: ...           │  │  文档内容（差异高亮）   │   │
│                      │  │                      │   │
│                      │  └──────────────────────┘   │
├──────────────────────┴──────────────────────────────┤
│  [输入框]                                [发送]     │
└─────────────────────────────────────────────────────┘
```

### 5.2 对话机制

| 特性 | 描述 |
|------|------|
| 头脑风暴模式 | AI 以讨论确认的方式与用户交互，提出建议后需用户确认才修改文档 |
| 建议卡片 | AI 的每条建议以卡片形式展示，包含建议内容和「采纳」「拒绝」按钮 |
| 实时预览 | 右侧预览区实时显示采纳建议后的文档效果 |
| 对话持久化 | 关闭弹窗后对话记录保留，再次打开可继续对话 |

### 5.3 版本管理

每次用户采纳AI建议后，自动生成一个新版本：

| 功能 | 描述 |
|------|------|
| 版本链 | 初始版本(v1) → 采纳建议(v2) → 采纳建议(v3) → ... |
| 版本选择器 | 下拉菜单可切换查看任意历史版本 |
| 差异高亮 | 预览区逐行展示，与上一版本对比高亮 |
| 恢复版本 | 点击「恢复此版本」将选中版本内容写回主文档，同时生成新版本记录 |
| 高亮规则 | 🟢 新增行（绿底）、🟡 修改行（黄底）、🔴 删除行（红底+删除线） |

### 5.4 对话记录存储

| 特性 | 描述 |
|------|------|
| 会话内保留 | 关闭弹窗后再打开，历史对话记录仍然存在 |
| 新建需求清空 | 开始新的需求标准化流程时清空对话记录 |

## 7. 草稿自动保存

### 7.1 保存机制

| 特性 | 描述 |
|------|------|
| 存储方式 | 浏览器 localStorage |
| 触发时机 | 内容变更后 3 秒防抖自动保存 |
| 立即保存 | 步骤切换、页面关闭时立即保存 |
| 保存范围 | 完整工作状态：步骤位置、输入内容、标准化文档、拆分需求、对话记录、版本链 |

### 7.2 草稿恢复

| 特性 | 描述 |
|------|------|
| 检测提示 | 页面加载时检测到草稿，显示琥珀色提示条「检测到未完成的草稿，保存于 xxx」 |
| 恢复操作 | 点击「恢复草稿」恢复全部工作状态 |
| 丢弃操作 | 点击「丢弃」清除草稿数据 |
| 自动清除 | 新建需求或加载历史记录时自动清除草稿 |

### 7.3 保存状态指示器

每个步骤标题旁显示实时保存状态：

| 状态 | 图标 | 说明 |
|------|------|------|
| 未保存 | 🟡 橙色圆点 | 内容已修改，3秒后自动保存 |
| 已保存 | ✅ 绿色勾 | 内容已保存至本地 |
| 保存失败 | 🔴 红色文字 | 保存异常（如存储空间不足） |

## 8. 文档导出

### 8.1 导出入口

步骤2底部操作栏的「导出」下拉按钮，点击展开格式选择菜单。

### 8.2 支持格式

| 格式 | 文件名 | 实现方式 |
|------|--------|----------|
| Markdown (.md) | 需求规格说明书_YYYY-MM-DD.md | Blob 直接下载 |
| Word 文档 (.docx) | 需求规格说明书_YYYY-MM-DD.docx | docx 库生成 |

### 8.3 DOCX 样式映射

| Markdown 语法 | DOCX 样式 |
|---------------|-----------|
| `# 标题` | Heading 1（36pt 加粗） |
| `## 标题` | Heading 2（28pt 加粗） |
| `### 标题` | Heading 3（24pt 加粗） |
| `> 引用` | 斜体灰色缩进段落 |
| 空行 | 段落间距 |
| 普通文本 | 22pt 正文 |

## 9. 历史记录

### 9.1 记录管理

| 功能点 | 描述 |
|--------|------|
| 自动记录 | 每次完成需求标准化流程后，自动新增一条历史记录 |
| 完整数据保留 | 保留原始需求、标准化文档、拆分需求、测试点、测试用例 |
| 查看记录 | 点击历史记录项加载完整数据，覆盖当前工作区 |
| 数据维护 | 支持从历史记录入口进行数据编辑和重新生成 |

### 9.2 与新建流程的关系

- 默认进入模块时，显示新建流程入口（三个步骤）
- 选中历史记录后，当前工作区被覆盖
- 点击「+ 新建需求」可随时开始新的标准化流程

## 11. 数据模型

### 需求表 (requirements)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| title | String | 需求标题 |
| raw_content | Text | 原始需求内容 |
| standardized_content | Text | 标准化后的文档内容（Markdown） |
| split_requirements | JSON | 拆分后的需求列表 |
| status | Enum | 状态（待处理/已标准化/已拆分） |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 标准化版本表 (standardize_versions)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| version_number | Integer | 版本号 |
| content | Text | 该版本文档内容 |
| description | String | 版本描述（如"初始版本"、"采纳AI建议"） |
| created_at | DateTime | 创建时间 |

### 对话记录表 (chat_messages)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| role | String | 角色（user/assistant） |
| content | Text | 消息内容 |
| proposal | Text | AI建议内容（仅assistant角色） |
| confirmed | Boolean | 用户是否采纳（仅assistant角色） |
| created_at | DateTime | 创建时间 |
