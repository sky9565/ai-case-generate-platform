# 测试设计模块 - 产品需求文档

## 1. 模块概述

### 1.1 模块定位
测试设计模块是智能测试用例平台的核心模块，负责将需求标准化模块产出的拆分需求，通过AI自动生成测试点和测试用例，并以脑图形式直观展示需求-测试点-测试用例的层级关系，支持用户手动编辑、AI辅助调整和导出。

### 1.2 核心流程

```
选择需求 → 查看脑图 → 快速生成/AI调整/手动操作 → 导出Excel
```

### 1.3 模块价值
- **可视化**：脑图形式直观展示需求→测试点→测试用例的层级关系
- **智能化**：AI自动生成测试点和测试用例，支持多轮对话式调整
- **可管控**：标记保留机制确保关键测试点不被AI误替换
- **可追溯**：版本管理保留AI调整的历史快照，支持回退

## 2A. 工具栏功能详细交互

### 2A.1 知识库开关

| 功能点 | 描述 |
|--------|------|
| 控件类型 | Toggle 开关 |
| 默认状态 | 关闭 |
| 开启效果 | 开关背景变为蓝色，生成时传递 `useKnowledgeBase: true` 参数 |
| 关闭效果 | 开关背景为灰色，生成时不启用知识库 |
| 交互方式 | 点击切换开关状态 |
| 影响范围 | 仅影响「快速生成」功能，不影响已生成的数据 |
| 视觉样式 | 关闭：灰色圆点在左；开启：蓝色背景、白色圆点在右 |

### 2A.2 快速生成

| 功能点 | 描述 |
|--------|------|
| 触发方式 | 点击工具栏"快速生成"按钮 |
| 前置条件 | 已选中一个需求 |
| 按钮禁用 | 未选中需求或正在生成中时按钮置灰不可点击 |
| 生成流程 | 1. 调用生成接口，传入 `useKnowledgeBase` 参数<br>2. 获取 `taskId`，启动轮询<br>3. 每 2 秒轮询任务状态接口<br>4. 更新进度条和进度描述文本<br>5. 任务完成后刷新脑图数据 |
| 按钮文案 | 默认："快速生成"；生成中："生成中..." |
| 按钮图标 | 默认：闪电图标；生成中：旋转加载图标 |
| 错误处理 | 接口失败时弹出提示，重置生成状态 |
| 完成后 | 更新需求状态为"已完成"，2秒后自动隐藏进度条 |

**进度条交互**：

| 功能点 | 描述 |
|--------|------|
| 显示时机 | 仅在生成中时显示，位于工具栏下方 |
| 进度百分比 | 左侧显示进度描述文本，右侧显示百分比数字 |
| 进度条样式 | 蓝色进度条，圆角，高度 1.5，带过渡动画（duration 500ms ease-out） |
| 进度描述 | 由后端返回的 `progressText` 字段驱动，如"正在分析需求结构..."、"正在生成测试点：用户名输入验证"等 |
| 完成后 | 进度条显示 100%，描述文本显示"生成完成"，2秒后自动隐藏 |

**轮询机制**：

| 功能点 | 描述 |
|--------|------|
| 轮询间隔 | 2 秒 |
| 轮询条件 | `currentTaskId` 存在且 `isGenerating` 为 true |
| 停止条件 | 任务状态为 `completed`、`failed`、`cancelled` 时停止轮询 |
| 任务完成 | 停止轮询 → 更新需求状态 → 重新加载脑图数据 → 2秒后隐藏进度条 |
| 任务失败 | 停止轮询 → 弹出错误提示 → 重置生成状态 |
| 任务取消 | 停止轮询 → 重置生成状态 |

### 2A.3 取消生成

| 功能点 | 描述 |
|--------|------|
| 触发方式 | 点击工具栏"取消"按钮 |
| 显示时机 | 仅在生成中时显示 |
| 按钮样式 | 红色文字按钮，hover 时显示浅红色背景 |
| 交互流程 | 1. 调用取消任务接口<br>2. 成功后停止轮询<br>3. 重置生成状态（进度、进度文本、任务ID） |
| 取消后 | 进度条消失，按钮恢复为"快速生成"，需求状态保持不变 |
| 无确认弹窗 | 点击取消直接执行，无需二次确认 |

### 2A.4 导出Excel

| 功能点 | 描述 |
|--------|------|
| 触发方式 | 点击工具栏"导出Excel"按钮 |
| 前置条件 | 已选中一个需求 |
| 按钮禁用 | 未选中需求或正在导出中时按钮置灰不可点击 |
| 按钮文案 | 默认："导出Excel"；导出中："导出中..." |
| 按钮图标 | 默认：下载图标；导出中：旋转加载图标 |
| 按钮样式 | 白色背景、灰色边框、灰色文字，hover 时浅灰背景 |
| 导出流程 | 1. 获取当前脑图数据<br>2. 解析脑图数据构建导出行<br>3. 生成 Excel 文件（含单元格合并和列宽设置）<br>4. 浏览器自动下载文件 |
| 文件命名 | `{需求标题}_测试用例.xlsx` |
| 无数据提示 | 若无脑图数据，弹出提示"暂无脑图数据可导出" |
| 导出失败 | 弹出提示"导出失败，请重试" |
| 防重复点击 | 导出中按钮禁用，防止重复触发 |

### 2.4 底部缩放控制栏

| 功能点 | 描述 |
|--------|------|
| 提示文案 | "提示：右键节点打开菜单 \| 双击用例编辑 \| 滚轮缩放" |
| 缩小按钮 | 点击缩小视图，每次缩小 10%，最小缩放 30% |
| 缩放比例 | 显示当前缩放百分比（如 100%），实时更新 |
| 放大按钮 | 点击放大视图，每次放大 10%，最大缩放 200% |
| 适应画布按钮 | 自动调整缩放使脑图内容适配画布 |

**缩放控制详细交互**：

| 功能点 | 描述 |
|--------|------|
| 缩放范围 | 30% ~ 200%（即 scale 值 0.3 ~ 2.0） |
| 缩放步进 | 每次点击放大/缩小按钮，scale 变化 0.1 |
| 缩小按钮 | `setScale(Math.max(current - 0.1, 0.3))`，到达 30% 后不再缩小 |
| 放大按钮 | `setScale(Math.min(current + 0.1, 2))`，到达 200% 后不再放大 |
| 适应画布 | 调用 `mindMap.view.fit()` 自动计算最佳缩放比例 |
| 比例显示 | 实时显示 `Math.round(scale * 100)%`，如 100%、150%、30% |
| 滚轮缩放 | 鼠标滚轮也可缩放视图，由 simple-mind-map 内置支持 |
| 按钮样式 | 灰色图标按钮，hover 时变深灰色并显示浅灰背景 |

## 3. 脑图结构

### 3.1 四级节点结构

| 层级 | 节点类型 | 说明 | 视觉样式 |
|------|----------|------|----------|
| 根节点 | 入口需求 | 需求文本简称 | 蓝色渐变背景圆角卡片，白色文字 |
| 二级节点 | 需求 | 拆分后的单个需求 | 浅蓝背景，蓝色文字，蓝色边框 |
| 三级节点 | 测试点 | 根据需求提取的测试点 | 浅绿背景，绿色文字，绿色边框；标记保留时左上角显示📌图标 |
| 四级节点 | 测试用例 | 根据测试点生成的测试用例 | 白色背景，灰色文字，灰色边框；左侧显示正例/反例标签 |

### 3.2 节点来源标识

| 来源 | 标识方式 |
|------|----------|
| AI 生成 | 节点数据中 `_source: 'AI'`，由后端标记 |
| 人工添加 | 节点数据中 `_source: '人工'`，添加时自动标记 |

### 3.3 节点状态图标

| 状态 | 图标 | 说明 |
|------|------|------|
| 待生成 | 灰色圆点 | 需求节点或测试点节点尚未生成子节点 |
| 生成中 | 蓝色旋转图标 | 正在AI生成中 |
| 已完成 | 绿色勾选 | 已完成子节点生成 |

```

│  添加测试点                         │
│  在「需求名称」节点下添加新的测试点    │
├────────────────────────────────────┤
│                                    │
│  测试点名称 *                       │
│  ┌──────────────────────────────┐ │
│  │ 请输入测试点名称              │ │
│  └──────────────────────────────┘ │
│                                    │
│  测试点描述                         │
│  ┌──────────────────────────────┐ │
│  │                              │ │
│  │ 请输入测试点描述（选填）       │ │

| 字段 | 必填 | 说明 |

| 功能点 | 描述 |

## 10. AI调整

AI调整功能支持两种层级：
- **需求级**：对需求节点下的测试点进行调整，标记保留作用于测试点
- **测试点级**：对测试点节点下的测试用例进行调整，标记保留作用于测试用例

两种层级的核心交互逻辑一致，区别仅在于操作的数据层级不同。

### 10.1 弹窗布局

```
┌──────────────────────────────────────────────────────────────┐
│  ⚡ AI 调整                                       [✕ 关闭]   │

│  （需求标签蓝色，测试点标签绿色）                               │
├───────────────────────────┬──────────────────────────────────┤
│  对话区（左 50%）          │  脑图预览区（右 50%）              │
│  bg-gray-50               │  bg-[#f0f4f8]                    │
│                           │  ┌──────────────────────────┐   │
│  [对话消息列表]            │  │ 版本选择器  [恢复此版本]   │   │
│                           │  ├──────────────────────────┤   │
│  AI: 我建议将...          │  │ 提示：右键测试点节点可标记  │   │
│  [采纳] [不采纳]          │  │ 标记保留：N 个测试点       │   │

### 10.7 采纳AI建议流程

```
用户点击"采纳"
    ↓
收集当前标记保留的节点列表
  - 需求级：收集标记的测试点名称列表
  - 测试点级：收集标记的测试用例名称列表
    ↓
获取当前主脑图数据快照
    ↓
显示"正在应用AI调整，保留标记的XX..."提示
  - 需求级：保留标记的测试点
  - 测试点级：保留标记的测试用例
    ↓
调用 applyAiAdjustment 接口
  - 传入 currentMindMapData（当前脑图数据）
  - 传入 markedTestPointTexts（标记保留的节点名称列表）
  - 传入 nodeType（节点类型：requirement / testPoint）
    ↓
后端处理逻辑：
  - 需求级：保留标记的测试点不变，对非标记的测试点进行AI调整
  - 测试点级：保留标记的测试用例不变，对非标记的测试用例进行AI调整
    ↓
返回调整后的脑图数据 + 统计信息
    ↓
前端更新主脑图数据
    ↓
显示调整结果统计（新增N个 / 保留N个 / 移除N个）
  - 需求级：统计测试点数量
  - 测试点级：统计测试用例数量
    ↓
自动保存新版本
    ↓
预览区脑图更新
```

### 15.1 采纳AI建议接口详情

| 属性 | 值 |
|------|-----|
| URL | `/api/v1/test-design/ai-adjust/sessions/{sessionId}/apply` |
| Method | `POST` |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| currentMindMapData | object | 是 | 当前主脑图完整数据快照 |
| markedTestPointTexts | string[] | 是 | 标记保留的节点名称列表（需求级为测试点名称，测试点级为测试用例名称） |
| nodeType | string | 是 | 节点类型：`requirement`（需求级，调整测试点）/ `testPoint`（测试点级，调整测试用例） |

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| data.adjustedMindMapData | object | AI调整后的完整脑图数据 |
| data.addedCount | integer | 新增节点数量（需求级为测试点数，测试点级为测试用例数） |
| data.removedCount | integer | 移除节点数量 |
| data.preservedCount | integer | 保留的标记节点数量 |

**处理逻辑**：
1. 接收当前脑图数据、标记保留列表和节点类型
2. 根据 nodeType 决定调整层级：
   - `requirement`：遍历需求节点，对测试点进行分类和调整
   - `testPoint`：遍历测试点节点，对测试用例进行分类和调整
3. 对标记保留的节点保持不变
4. 对非标记的节点执行AI调整（新增/修改/删除）
5. 返回调整后的完整脑图数据和统计信息
