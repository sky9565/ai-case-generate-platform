# 智能测试用例平台 - 完整需求文档

## 1. 项目概述

### 1.1 项目名称
**智能测试用例平台**

### 1.2 项目目标
为银行软件测试人员提供AI驱动的测试用例生成平台，通过上传需求文档或输入一句话需求，自动完成需求文档标准化、需求拆分、测试点提取和测试用例生成，并以脑图形式直观展示。

### 1.3 核心价值
- **提高效率**：AI自动生成测试点和测试用例，大幅减少手工编写时间
- **质量保障**：基于标准化需求文档生成，确保测试覆盖全面
- **可视化管理**：脑图形式直观展示需求-测试点-测试用例的层级关系

## 2. 功能需求

### 2.1 登录模块

| 功能点 | 描述 |
|--------|------|
| 用户注册 | 支持用户名密码注册 |
| 用户登录 | 支持用户名密码登录 |
| 数据隔离 | 登录后仅能查看自己的数据，纯个人使用模式 |
| 登录落地页 | 登录成功后默认进入"需求标准化"模块 |

### 2.2 需求标准化模块

#### 2.2.1 需求录入
| 功能点 | 描述 |
|--------|------|
| 文本输入 | 支持直接输入一句话需求 |
| 文档上传 | 支持上传docx、xlsx格式文档 |
| 二选一模式 | 文本或文档两者只能选择其一 |

#### 2.2.2 文档标准化助手
| 功能点 | 描述 |
|--------|------|
| 标准化生成 | 点击按钮调用大模型将不规范需求转换为标准md文档 |
| 预览编辑 | 标准化后的文档在"标准文档预览"区域展示，支持用户编辑 |
| 多轮对话 | 支持调起嵌入页面的对话框进行多轮对话，调整输出内容 |
| 对话历史 | 保留对话历史记录 |

#### 2.2.3 需求拆分
| 功能点 | 描述 |
|--------|------|
| 自动拆分 | 将标准化文档拆分为单个需求 |
| 手动编辑 | 支持对拆分后的需求进行编辑 |
| 手动删除 | 支持删除不需要的需求 |
| 手动新增 | 支持手动添加新需求 |

#### 2.2.4 历史记录
| 功能点 | 描述 |
|--------|------|
| 自动记录 | 每次处理完一个需求后，自动新增一条历史记录 |
| 完整数据保留 | 保留完整数据：原始需求、拆分后的需求、测试点、测试用例 |
| 查看记录 | 支持查看历史记录列表 |
| 数据维护 | 支持从历史记录入口进行数据维护（编辑、重新生成等） |

### 2.3 测试设计模块（脑图展示）

#### 2.3.1 脑图结构（4级）
| 层级 | 节点类型 | 说明 |
|------|----------|------|
| 根节点 | 入口需求 | 需求文本或需求文档，显示简称 |
| 二级节点 | 需求 | 拆分后的单个需求 |
| 三级节点 | 测试点 | 根据需求提取的测试点 |
| 四级节点 | 测试用例 | 根据测试点生成的测试用例 |

#### 2.3.2 脑图操作功能
| 功能点 | 描述 |
|--------|------|
| 展开/折叠 | 点击节点展开或折叠子节点 |
| 缩放 | 鼠标滚轮或按钮缩放脑图视图 |
| 拖拽移动视图 | 按住鼠标拖拽空白区域移动视图 |
| 右键菜单 | 右键点击节点弹出操作菜单 |
| 双击编辑 | 双击节点文字进行编辑 |

#### 2.3.3 节点右键菜单
| 节点类型 | 支持操作 |
|----------|----------|
| 需求节点 | 添加测试点、AI调整 |
| 测试点节点 | 编辑、删除、添加测试用例、AI调整 |
| 测试用例节点 | 编辑、删除 |

#### 2.3.4 AI调整功能
| 功能点 | 描述 |
|--------|------|
| 标记保留 | 使用图钉图标放在节点前面作为标记 |
| 标记显示 | 仅通过图钉图标区分，无其他视觉区分 |
| 标记生效范围 | 测试点或用例被标记后，在"AI调整"的多轮对话中都生效 |
| 默认替换 | 未标记的数据在AI调整时被替换 |
| 多轮对话 | 支持调起嵌入页面的对话框进行多轮对话调整 |

#### 2.3.5 异步任务管理
| 功能点 | 描述 |
|--------|------|
| 快速生成 | 一键调起所有需求生成测试点及用例的任务 |
| 串行执行 | 需求按顺序串行生成，避免资源过度占用 |
| 深度优先 | 单个需求→多个测试点→多个测试用例，再到下一需求 |
| 任务取消 | 支持在生成过程中取消任务 |
| 实时进度条 | 显示整体生成进度（如"正在处理需求3/10，已完成30%"） |
| 节点状态图标 | 每个需求节点及测试点节点显示状态（待生成/生成中/已完成） |

#### 2.3.6 操作栏
| 功能点 | 描述 |
|--------|------|
| 快速生成按钮 | 一键触发生成所有测试点及用例 |
| 知识库开关 | 开启后生成时可检索知识库内容作为参考 |
| 导出按钮 | 导出测试用例为Excel格式 |

#### 2.3.7 批量操作功能
| 功能点 | 描述 |
|--------|------|
| 批量删除 | 支持批量删除测试点或测试用例 |
| 批量标记 | 支持批量标记测试点或测试用例为"保留"状态 |

#### 2.3.8 导出功能
| 功能点 | 描述 |
|--------|------|
| 导出Excel | 将测试用例导出为Excel格式 |
| 表头字段 | caseName、caseProperty、preCondition、name、description、stepExpectedResult |
| 数据展示 | steps的3个子字段各占一列，每个步骤占一行 |
| 行合并 | 若用例有多个步骤，则该用例的caseName/caseProperty/preCondition单元格合并展示 |

**Excel导出示例**：

| caseName | caseProperty | preCondition | name | description | stepExpectedResult |
|----------|--------------|---------------|------|-------------|-------------------|
| 用户正常登录系统 | 正例 | 用户已注册并拥有有效的账号和密码 | 输入用户名 | 在登录页输入正确的用户名 | 用户名输入框显示输入的用户名 |
| (合并) | (合并) | (合并) | 输入密码 | 在登录页输入正确的密码 | 密码输入框显示输入的密码 |
| (合并) | (合并) | (合并) | 点击登录按钮 | 点击登录按钮 | 系统跳转至用户主页 |
| 用户登录失败 | 反例 | 用户已注册但密码错误 | 输入用户名 | 在登录页输入用户名 | 用户名输入框显示输入的用户名 |
| (合并) | (合并) | (合并) | 点击登录按钮 | 点击登录按钮 | 系统提示密码错误 |

#### 2.3.9 节点来源标识
| 功能点 | 描述 |
|--------|------|
| 测试点来源 | 区分测试点是人工添加还是AI生成 |
| 测试用例来源 | 区分测试用例是人工添加还是AI生成 |
| 标识方式 | 通过标签进行区分（如"AI"或"人工"标签） |

### 2.4 知识库模块

#### 2.4.1 文档管理
| 功能点 | 描述 |
|--------|------|
| 文档上传 | 支持用户上传文档到知识库 |

#### 2.4.2 知识召回设置
| 功能点 | 描述 |
|--------|------|

### 2.5 外部AI服务对接
| 功能点 | 描述 |
|--------|------|
| 后端硬编码 | 直接在后端代码中配置AI服务，不做交互界面 |
| 默认模型 | 后端默认配置一个模型（如OpenAI GPT-4），可人工后续修改 |
| 灵活扩展 | 预留多模型扩展能力，后续可对接其他大模型 |

### 2.6 测试用例模板

| 字段 | 英文名 | 说明 |
|------|--------|------|
| 测试用例名称 | caseName | 用例的唯一标识名称 |
| 用例类型 | caseProperty | 标记该用例是正例还是反例 |
| 前置条件 | preCondition | 执行该用例前需要满足的条件 |
| 步骤 | steps | 步骤列表，每个步骤包含步骤名字、步骤描述、步骤预期结果 |

**steps字段结构**：
| 子字段 | 英文名 | 说明 |
|--------|--------|------|
| 步骤名字 | name | 步骤的名称 |
| 步骤描述 | description | 步骤的具体操作描述 |
| 步骤预期结果 | stepExpectedResult | 该步骤执行后期望的结果 |

**测试用例JSON示例**：
```json
{
  "caseName": "用户正常登录系统",

    {
      "name": "输入用户名",

    },
    {
      "name": "输入密码",
      "description": "在登录页输入正确的密码",
      "stepExpectedResult": "密码输入框显示输入的密码"
    },
    {
      "name": "点击登录按钮",
      "description": "点击登录按钮",
      "stepExpectedResult": "系统跳转至用户主页"
    }
  ]
}
```

## 3. 非功能需求

### 3.1 性能需求
- 单条需求生成测试点响应时间 ≤ 10秒
- 批量生成10条需求的测试点和用例总时间 ≤ 60秒
- 脑图渲染100个节点响应时间 ≤ 2秒
- Excel导出1000条测试用例时间 ≤ 10秒
- 历史记录查询响应时间 ≤ 3秒

### 3.2 安全需求
- 用户密码采用加密存储
- 登录会话超时时间 ≤ 30分钟
- 所有API请求需要身份认证
- AI服务API密钥加密存储

### 3.3 可用性需求
- 系统可用性 ≥ 99.5%
- 页面响应时间 ≤ 3秒

### 3.4 可扩展性需求
- 支持对接多种外部AI服务
- 支持扩展新的文档格式上传

## 4. 用户流程

```
用户登录 → 进入需求标准化模块
    ↓
需求录入（文本或上传文档）
    ↓
点击"文档标准化助手"
    ↓
大模型生成标准文档 → 用户确认/编辑
    ↓
需求拆分 → 生成二级需求节点
    ↓
进入测试设计模块（脑图展示）
    ↓
点击"快速生成"或单个需求"AI调整"
    ↓
异步生成测试点和测试用例（深度优先、串行执行）
    ↓
实时展示进度和节点状态
    ↓
生成完成 → 用户可编辑/删除/添加/批量操作测试点和用例
    ↓
点击"导出" → 导出为Excel文件
```

## 5. 技术架构

### 5.1 架构风格
- 前后端分离架构

### 5.2 前端技术栈
- Vue 2 + Vue Router + Vuex
- Webpack
- 脑图组件（需选型）
- Excel导出库（如xlsx或exceljs）

### 5.3 后端技术栈
- Python 3.12
- FastAPI
- LangGraph（智能体编排）
- Milvus（向量数据库）
- OpenAI API/其他大模型API（后端硬编码配置）

## 7. 数据模型（概要）

### 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| username | String | 用户名 |
| password | String | 加密密码 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 需求表 (requirements)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| title | String | 需求标题 |
| content | Text | 需求内容 |
| status | Enum | 状态（待处理/已标准化/已拆分） |

| content | String | 测试点内容 |
| status | Enum | 状态（待生成/生成中/已完成） |

### 历史记录表 (history_records)
| 字段 | 类型 | 说明 |
|------|------|------|

| test_points | JSON | 测试点列表 |
| test_cases | JSON | 测试用例列表 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |
