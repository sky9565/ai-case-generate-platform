# 用户认证模块 - 产品需求文档

## 1. 模块概述

### 1.1 模块定位
用户认证模块是智能测试用例平台的基础安全模块，负责用户身份验证、访问控制和数据隔离，确保平台安全性和用户数据的私密性。

### 1.2 核心价值
- **安全保障**：提供安全的用户认证机制，保护用户数据
- **数据隔离**：实现用户数据完全隔离，确保个人隐私
- **简化体验**：简洁的登录注册流程，降低用户使用门槛
- **状态管理**：完善的用户会话管理和认证状态维护

### 1.3 设计原则
1. **简洁性**：去除不必要的功能，保持界面简洁
2. **安全性**：采用安全的认证机制和密码存储
3. **可用性**：提供友好的错误提示和用户引导
4. **一致性**：与平台整体设计风格保持一致

## 2. 功能需求

### 2.1 用户登录

#### 2.1.1 登录页面
| 功能点 | 描述 | 状态 |
|--------|------|------|
| 用户名输入 | 支持用户名输入，3-20位字符 | ✅ 已实现 |
| 密码输入 | 支持密码输入，至少6位字符 | ✅ 已实现 |
| 密码显示切换 | 支持密码显示/隐藏切换 | ✅ 已实现 |
| 记住我功能 | 支持记住用户名，下次自动填充 | ✅ 已实现 |
| 表单验证 | 客户端表单验证，实时错误提示 | ✅ 已实现 |
| 加载状态 | 登录过程中的加载状态显示 | ✅ 已实现 |
| 错误提示 | 友好的错误提示信息 | ✅ 已实现 |
| 注册链接 | 跳转到注册页面的链接 | ✅ 已实现 |

#### 2.1.2 登录流程
1. 用户访问平台，自动跳转到登录页面
2. 输入用户名和密码
3. 可选：勾选"记住我"选项
4. 点击登录按钮
5. 系统验证用户身份
6. 验证成功：跳转到需求标准化模块
7. 验证失败：显示错误提示

#### 2.1.3 验证规则
- **用户名**：必填，不能为空
- **密码**：必填，长度至少6位
- **记住我**：可选，勾选后下次自动填充用户名

### 2.2 用户注册

#### 2.2.1 注册页面
| 功能点 | 描述 | 状态 |
|--------|------|------|
| 用户名输入 | 支持用户名输入，3-20位字符 | ✅ 已实现 |
| 密码输入 | 支持密码输入，至少6位字符 | ✅ 已实现 |
| 确认密码 | 支持密码确认输入 | ✅ 已实现 |
| 密码显示切换 | 支持密码显示/隐藏切换 | ✅ 已实现 |
| 表单验证 | 客户端表单验证，实时错误提示 | ✅ 已实现 |
| 加载状态 | 注册过程中的加载状态显示 | ✅ 已实现 |
| 错误提示 | 友好的错误提示信息 | ✅ 已实现 |
| 登录链接 | 跳转到登录页面的链接 | ✅ 已实现 |

#### 2.2.2 注册流程
1. 用户点击"立即注册"链接
2. 输入用户名、密码、确认密码
3. 点击注册按钮
4. 系统验证用户名唯一性
5. 验证成功：自动登录并跳转到需求标准化模块
6. 验证失败：显示错误提示

#### 2.2.3 验证规则
- **用户名**：必填，不能为空，用户名唯一
- **密码**：必填，长度至少6位
- **确认密码**：必填，必须与密码一致

### 2.3 用户认证状态管理

#### 2.3.1 认证状态
| 功能点 | 描述 | 状态 |
|--------|------|------|

| 路由守卫 | 保护需要认证的路由 | ✅ 已实现 |
| 自动跳转 | 未登录用户访问受保护路由时跳转到登录页 | ✅ 已实现 |

#### 2.3.2 导航栏状态
| 功能点 | 描述 | 状态 |
|--------|------|------|
| 用户信息显示 | 显示当前登录用户名 | ✅ 已实现 |
| 退出登录 | 支持退出登录功能 | ✅ 已实现 |
| 状态同步 | 实时更新用户认证状态 | ✅ 已实现 |

#### 2.3.3 退出登录
1. 用户点击导航栏的"退出登录"按钮
2. 清除本地存储的Token
3. 清除Vuex中的用户状态
4. 跳转到登录页面

### 2.4 数据隔离

#### 2.4.1 隔离机制
| 功能点 | 描述 | 状态 |
|--------|------|------|
| 用户数据隔离 | 每个用户只能访问自己的数据 | ✅ 已实现 |
| 基于Token认证 | 所有API请求需要携带Token | ✅ 已实现 |
| 个人使用模式 | 纯个人使用，不支持团队协作 | ✅ 已实现 |

#### 2.4.2 实现方式
- 前端：所有API请求自动携带Token
- 后端：根据Token识别用户，返回对应用户数据
- 存储：用户数据按用户ID隔离存储

## 4. 技术实现

### 4.1 前端技术栈
| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 2 | 2.7.14 | 前端框架 |
| Vue Router | 3.6.5 | 路由管理 |
| Vuex | 3.6.2 | 状态管理 |
| Tailwind CSS | 3.4.14 | 样式框架 |
| Axios | 1.6.7 | HTTP客户端 |

### 4.2 核心组件

#### 4.2.1 Login.vue
- **功能**：用户登录页面
- **特性**：表单验证、记住我功能、错误处理
- **状态管理**：本地状态 + Vuex状态同步

#### 4.2.2 Register.vue
- **功能**：用户注册页面
- **特性**：表单验证、密码确认、错误处理
- **状态管理**：本地状态 + Vuex状态同步

#### 4.2.3 TopNav.vue
- **功能**：导航栏组件
- **特性**：用户信息显示、退出登录功能
- **状态管理**：从Vuex获取用户状态

### 4.3 状态管理

#### 4.3.1 Vuex Store结构
```javascript
state: {
  user: null,           // 用户信息
  token: null,          // 认证Token
  isLoggedIn: false     // 登录状态
}
```

#### 4.3.2 状态同步
- **登录时**：Token存入localStorage，更新Vuex状态
- **退出时**：清除localStorage，重置Vuex状态
- **初始化时**：从localStorage恢复登录状态

### 4.4 路由配置

#### 4.4.1 路由结构
```javascript
const routes = [
  { path: '/login', component: Login, meta: { public: true } },
  { path: '/register', component: Register, meta: { public: true } },
  { path: '/standardization', component: Standardization, meta: { requiresAuth: true } },
  { path: '/test-design', component: TestDesign, meta: { requiresAuth: true } },
  { path: '/knowledge-base', component: KnowledgeBase, meta: { requiresAuth: true } },
  { path: '/', redirect: '/login' }
]
```

#### 4.4.2 路由守卫
- 访问需要认证的路由时检查Token
- 未登录用户自动跳转到登录页面
- 已登录用户访问登录/注册页面时跳转到首页

### 4.5 API接口

#### 4.5.1 接口规范
- **基础路径**：`/api/auth`
- **请求格式**：JSON
- **响应格式**：统一JSON格式
- **认证方式**：Bearer Token

#### 4.5.2 Mock接口（开发阶段）
- **登录接口**：`POST /api/auth/login`
- **注册接口**：`POST /api/auth/register`
- **用户数据**：内存存储，重启后重置

## 6. 用户流程

### 6.1 新用户流程
```
访问平台 → 跳转到登录页 → 点击"立即注册"
    ↓
进入注册页 → 输入用户名、密码、确认密码
    ↓

    ↓
自动登录 → 跳转到需求标准化模块
```

### 6.2 现有用户流程
```
访问平台 → 跳转到登录页 → 输入用户名密码
    ↓
点击登录 → 系统验证 → 登录成功
    ↓
跳转到需求标准化模块 → 开始使用平台
```

### 6.3 退出流程
```
点击导航栏"退出登录" → 清除Token和用户状态
    ↓
跳转到登录页 → 可重新登录或注册
```

## 10. 附录

### 10.1 设计决策说明

#### 10.1.1 功能简化决策
- **移除邮箱输入**：简化注册流程，降低用户输入负担
- **移除服务条款**：内部使用平台，简化法律合规要求
- **移除第三方登录**：保持平台简洁性，减少外部依赖
- **移除忘记密码**：简化密码管理，降低维护成本

#### 10.1.2 技术选型决策
- **Vue 2**：团队熟悉，生态成熟，满足项目需求
- **Tailwind CSS**：快速开发，设计一致性，响应式支持
- **Vuex**：状态管理需求简单，Vue生态集成好
- **LocalStorage**：Token存储简单，满足当前需求
