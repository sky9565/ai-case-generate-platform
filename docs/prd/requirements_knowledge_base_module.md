# 知识库模块 - 产品需求文档

## 1. 模块概述

### 1.1 模块定位
知识库模块是智能测试用例平台的辅助模块，负责管理用户上传的参考文档，通过文档切片和向量化入库，为测试用例生成提供领域知识检索能力。开启知识库后，AI在生成测试点和测试用例时可检索知识库中的相关内容作为参考，提升生成结果的专业性和准确性。

### 1.2 核心流程

```
上传文档 → 自动切片 → 向量化入库 → 知识就绪 → 测试设计模块引用
```

### 1.3 模块价值
- **专业性**：通过知识库注入领域知识，使AI生成的测试用例更贴合业务场景
- **可配置**：用户可调整召回参数和切片策略，灵活控制知识检索效果
- **可验证**：提供召回测试功能，用户可直观验证知识库的召回质量
- **透明性**：文档处理状态全流程可见，切片结果可查看

### 3.2 文档列表

#### 3.2.1 列表展示

| 信息项 | 描述 |

#### 3.2.2 文档处理状态

文档上传后自动进行切片和向量化处理，状态流转如下：

```
上传中 → 切片中 → 向量化中 → 已就绪
                                    ↓
                               处理失败
```

| 状态 | 标签样式 | 说明 |
|------|----------|------|
| 上传中 | 蓝色脉冲动画标签 | 文件正在上传到服务器 |
| 切片中 | 蓝色脉冲动画标签 | 系统正在对文档进行切片处理 |
| 向量化中 | 蓝色脉冲动画标签 | 切片内容正在向量化并存入Milvus |
| 已就绪 | 绿色标签 | 文档处理完成，可被知识召回检索 |
| 处理失败 | 红色标签 | 处理过程中出错，显示错误原因 |

**状态轮询机制**：
- 文档处于非终态（上传中/切片中/向量化中）时，前端每3秒轮询一次状态
- 到达终态（已就绪/处理失败）后停止轮询
- 页面切换回来时，对非终态文档立即触发一次状态查询

## 4. 知识召回设置

### 4.1 全局开关

知识召回设置区域顶部包含全局开关，控制知识召回功能的启用/禁用：

| 功能点 | 描述 |
|--------|------|
| 开关位置 | 知识召回设置区域标题栏右侧 |
| 默认状态 | 开启 |
| 功能说明 | 开关下方显示说明文字：「开启后，AI 生成测试点和测试用例时将自动检索知识库中的相关内容作为参考，提升生成结果的专业性和准确性」 |
| 禁用效果 | 关闭后，整个参数配置区域置灰且不可操作 |

### 4.2 参数配置

知识召回设置区域分为三组参数，以三列卡片形式展示：

#### 4.2.1 召回参数

| 参数 | 英文名 | 类型 | 默认值 | 范围 | 说明 |
|------|--------|------|--------|------|------|
| 召回数量 | top_k | 滑块 | 5 | 1-20 | 每次召回返回的最大切片数量 |
| 相似度阈值 | score_threshold | 滑块 | 0.7 | 0-1（步长0.01） | 仅返回相似度分数大于等于此阈值的结果 |
| 启用知识召回 | enabled | 开关 | 开启 | 开/关 | 全局开关，关闭后生成测试用例时不检索知识库 |

**参数效果说明**（显示在滑块下方）：

| 参数 | 效果说明 |
|------|----------|
| 召回数量 (top_k) | 每次召回返回的最大切片数量。值越大，提供的参考知识越多，但可能引入不相关内容；值越小，结果越精准但可能遗漏有用信息 |
| 相似度阈值 | 仅返回相似度分数 ≥ 此阈值的结果。提高阈值可过滤低相关内容，但可能减少有效召回；降低阈值可获取更多结果，但可能引入噪声 |

#### 4.2.2 切片参数

| 参数 | 英文名 | 类型 | 默认值 | 范围 | 说明 |
|------|--------|------|--------|------|------|
| 切片大小 | chunk_size | 滑块 | 500 | 100-2000（步长50） | 每个切片的最大字符数 |
| 切片重叠度 | chunk_overlap | 滑块 | 50 | 0-500（步长10） | 相邻切片之间重叠的字符数 |

**参数效果说明**（显示在滑块下方）：

| 参数 | 效果说明 |
|------|----------|
| 切片大小 (chunk_size) | 每个切片的最大字符数。较大的切片保留更多上下文，但可能降低检索精度；较小的切片检索更精准，但可能丢失上下文关联 |
| 切片重叠度 (chunk_overlap) | 相邻切片之间重叠的字符数。增大重叠度可避免关键信息被截断在切片边界，但会增加存储和计算开销 |

**切片参数警告提示**：
- 参数区域底部显示黄色警告框：「修改切片参数后保存，将触发所有文档重新切片和向量化处理」
- 修改切片参数后，需要重新处理所有已上传的文档（切片+向量化）
- 修改时弹出确认提示「修改切片参数后需要重新处理所有文档，是否继续？」
- 确认后所有已就绪文档状态变为「切片中」，重新走切片→向量化流程

#### 4.2.3 召回策略

| 参数 | 英文名 | 类型 | 默认值 | 选项 | 说明 |
|------|--------|------|--------|------|------|
| 召回策略 | recall_strategy | 单选卡片 | hybrid | vector / hybrid | vector=纯向量召回，hybrid=混合召回（向量+关键词） |

**交互方式**：以单选卡片形式展示，每个选项包含标题、描述和详细说明，选中时高亮显示并展开详细说明。

**策略说明**：

| 策略 | 标签 | 描述 | 详细说明（选中时展示） | 推荐 |
|------|------|------|----------------------|------|
| 混合召回 (hybrid) | 混合召回 | 结合向量相似度和关键词匹配，兼顾语义理解和精确匹配 | 适合大多数场景，特别是包含专业术语、编号规则等需要精确匹配关键词的文档 | ✅ 推荐 |
| 纯向量召回 (vector) | 纯向量召回 | 仅基于语义相似度进行召回，理解意图而非字面匹配 | 适合自然语言描述为主的文档，能找到语义相近但用词不同的内容 | - |

## 5. 与测试设计模块的联动

### 5.1 知识库开关

| 功能点 | 描述 |
|--------|------|
| 开关位置 | 测试设计模块操作栏中的「知识库」开关 |
| 全局生效 | 开启后，所有需求的测试点和测试用例生成时都会检索知识库 |
| 默认状态 | 跟随知识库模块的「启用知识召回」设置 |
| 状态同步 | 知识库模块修改启用状态后，测试设计模块的开关同步更新 |
| 无文档提示 | 知识库无已就绪文档时，开关旁显示提示「知识库暂无可用文档」 |

### 5.2 生成时的知识检索

| 功能点 | 描述 |
|--------|------|
| 检索时机 | 生成测试点和测试用例时，若知识库开关开启且有已就绪文档，自动检索相关知识 |
| 检索参数 | 使用知识库模块中配置的召回参数（召回数量、相似度阈值、召回策略） |
| 参考注入 | 检索到的知识片段作为上下文注入到AI生成的Prompt中 |
| 透明性 | 生成过程中不展示具体的知识检索细节，用户无需关注 |

## 7. 数据模型

### 知识文档表 (knowledge_docs)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| name | String | 文件名 |
| size | BigInteger | 文件大小（字节） |
| format | String | 文件格式（docx/xlsx/pdf/txt/md） |
| status | Enum | 处理状态（uploading/slicing/vectorizing/ready/failed） |
| chunk_count | Integer | 切片数量 |
| error_message | Text | 失败原因（仅failed状态） |
| retry_count | Integer | 已重试次数 |
| file_path | String | 文件存储路径 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 知识切片表 (knowledge_chunks)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| doc_id | UUID | 关联文档 |
| chunk_index | Integer | 切片序号（从1开始） |
| content | Text | 切片文本内容 |
| length | Integer | 切片字符数 |
| vector_id | String | Milvus中的向量ID |
| created_at | DateTime | 创建时间 |

### 知识召回设置表 (knowledge_settings)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 关联用户（每用户一条记录） |
| top_k | Integer | 召回数量，默认5 |
| score_threshold | Float | 相似度阈值，默认0.7 |
| enabled | Boolean | 是否启用知识召回，默认true |
| chunk_size | Integer | 切片大小，默认500 |
| chunk_overlap | Integer | 切片重叠度，默认50 |
| recall_strategy | String | 召回策略（vector/hybrid），默认hybrid |
| updated_at | DateTime | 更新时间 |

## 8. 前端交互细节

### 8.1 文档卡片交互

| 交互 | 行为 |
|------|------|
| 悬停 | 卡片边框变蓝，显示操作按钮（查看、切片、删除） |
| 查看图标 | 点击打开文档内容查看弹窗 |
| 切片数量 | 点击打开切片详情弹窗（仅已就绪状态可点击） |
| 删除图标 | 点击弹出删除确认对话框 |
| 重试按钮 | 仅处理失败状态显示，点击重新处理 |
| 状态标签 | 不同状态不同颜色，处理中状态有脉冲动画 |

### 8.2 搜索交互

| 功能点 | 描述 |
|--------|------|
| 搜索防抖 | 输入关键词后延迟300ms触发搜索，避免频繁请求 |
| 实时过滤 | 根据关键词实时过滤文档列表 |
| 搜索范围 | 匹配文档名称 |

### 8.3 存储空间提示

| 功能点 | 描述 |
|--------|------|
| 展示位置 | 统计卡片区第4个卡片，显示「已用 500MB / 2GB」 |
| 进度条 | 用进度条可视化展示存储空间使用比例 |
| 颜色变化 | 使用率<70%蓝色，70%-90%黄色，>90%红色 |
| 上传前校验 | 上传前检查剩余空间是否足够 |

### 8.4 参数修改确认

| 场景 | 确认内容 |
|------|----------|
| 修改切片参数 | 「修改切片参数后需要重新处理所有文档，是否继续？」确认→保存并触发重处理，取消→恢复原值 |
| 删除文档 | 「确定要删除文档"xxx"吗？删除后相关切片和向量数据将一并清除。」 |
| 覆盖同名文件 | 上传区域显示黄色警告「已存在同名文件，是否覆盖？」+ 「覆盖」按钮，点击覆盖后移除原文件并继续上传 |

### 8.5 空状态

| 场景 | 展示内容 |
|------|----------|
| 无文档 | 居中显示空状态插图 + 「暂无文档，点击上传按钮添加」+ 上传按钮 |
| 搜索无结果 | 「未找到匹配的文档」 |
| 召回测试无结果 | 「未找到相关内容，请尝试调整查询文本或降低相似度阈值」 |

### 8.6 Toast 通知

| 场景 | 通知类型 | 通知内容 |
|------|----------|----------|
| 文档上传成功 | 成功 | 「文档上传成功，正在处理中...」 |
| 文档上传失败 | 错误 | 「文档上传失败：{错误原因}」 |
| 文档删除成功 | 成功 | 「文档已删除」 |
| 参数保存成功 | 成功 | 「知识召回设置已保存」 |
| 参数保存失败 | 错误 | 「设置保存失败，请重试」 |
| 文件格式错误 | 错误 | 「不支持的文件格式，请上传 docx/xlsx/pdf/txt/md 文件」 |
| 文件大小超限 | 错误 | 「文件大小超过50MB限制」 |
| 文档数量达上限 | 错误 | 「文档数量已达上限（100个），请删除后再上传」 |
| 存储空间不足 | 错误 | 「存储空间不足，请删除部分文档后重试」 |

### 8.7 参数滑块交互

| 功能点 | 描述 |
|--------|------|
| 滑块样式 | 使用范围滑块（range input），自定义主题色（召回参数蓝色、切片参数紫色） |
| 实时数值 | 滑块右侧实时显示当前值，以等宽字体+浅色背景标签形式展示 |
| 范围标注 | 滑块下方标注最小值和最大值 |
| 效果说明 | 每个滑块下方显示参数效果说明文字，帮助用户理解参数调整的影响 |
| 禁用状态 | 知识召回开关关闭时，所有滑块置灰且不可操作 |
